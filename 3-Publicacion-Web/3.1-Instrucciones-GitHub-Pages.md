# 3.1 · Publicar el dashboard en GitHub Pages

Objetivo: obtener un link público (`https://TU-USUARIO.github.io/japon-2027/`) para
compartir el dashboard con terceros. El archivo a subir es `index.html` (en esta misma
carpeta) — debe llamarse exactamente así para que cargue en la raíz.

---

## Opción A — Línea de comandos con GitHub CLI (recomendada)

Windows / PowerShell. Reemplaza `TU-USUARIO` por tu usuario de GitHub.

```powershell
# 1. Carpeta limpia del proyecto
cd $HOME
mkdir japon-2027
cd japon-2027

# 2. Copia el index.html de esta carpeta (ajusta la ruta de origen)
copy "RUTA\A\3-Publicacion-Web\index.html" .\index.html

# 3. Repo local
git init
git add index.html
git commit -m "Documento Maestro Japon 2027"
git branch -M main

# 4. Sesión de GitHub (solo la primera vez)
gh auth login

# 5. Crea el repo público y sube
gh repo create japon-2027 --public --source=. --remote=origin --push

# 6. Activa Pages en main /(root)
gh api --method POST /repos/TU-USUARIO/japon-2027/pages -f "source[branch]=main" -f "source[path]=/"
```

Link final: **https://TU-USUARIO.github.io/japon-2027/** (~1 min en levantar).

Requisitos: `git --version` y `gh --version`. Si falta gh: `winget install GitHub.cli`.

---

## Opción B — Sin gh (crear repo en la web)

1. En github.com: **+ → New repository**, nombre `japon-2027`, **Public**, Create.
2. **uploading an existing file** → arrastra `index.html` → **Commit changes**.
3. **Settings → Pages → Source: Deploy from a branch → main → /(root) → Save**.
4. Espera ~1 min y copia el link que aparece arriba.
